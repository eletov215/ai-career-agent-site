# AI Career Agent — руководство по работе с проектом

## 1. Главный принцип

Актуальная версия проекта всегда находится в GitHub.

ChatGPT используется для:

- анализа кода;
- подготовки изменений;
- исправления ошибок;
- создания новых файлов;
- формирования ZIP-архива;
- документирования проекта;
- подготовки инструкций для GitHub и Render.

## 2. Как передавать проект в ChatGPT

Если прямое подключение GitHub недоступно:

1. Открой репозиторий GitHub.
2. Нажми `Code`.
3. Выбери `Download ZIP`.
4. Загрузи ZIP в чат внутри проекта ChatGPT.
5. Напиши задачу и укажи, что архив является текущей версией.

Пример:

```text
Это актуальный архив проекта из GitHub.
Исправь поиск вакансий HeadHunter.
Не создавай app_fixed.py.
Верни архив с изменёнными файлами и список изменений.
```

## 3. Что должен сделать агент перед изменением

1. Распаковать архив.
2. Проверить структуру.
3. Определить, какой файл реально запускается.
4. Прочитать связанные файлы.
5. Не менять лишние файлы.
6. Сохранить исходную структуру проекта.
7. Проверить синтаксис изменённых Python-файлов.
8. Создать итоговый ZIP.

## 4. Что агент должен вернуть

После выполнения задачи агент должен вернуть:

- изменённые файлы;
- итоговый ZIP проекта;
- перечень изменённых файлов;
- краткое описание изменений;
- способ проверки;
- известные ограничения;
- предупреждение, если результат не проверен на реальном API.

## 5. Формат ответа после изменения

```text
Изменены файлы:
- app.py
- services/hh_provider.py

Что изменено:
- ...
- ...

Как загрузить:
1. ...
2. ...

Как проверить:
1. ...
2. ...

Ограничения:
- ...
```

## 6. Как обновлять GitHub через сайт

1. Открой репозиторий.
2. Перейди к нужному файлу.
3. Нажми значок редактирования.
4. Замени содержимое.
5. Нажми `Commit changes`.

Для нескольких файлов удобнее загрузить полный архив через GitHub Desktop или VS Code.

## 7. Как обновлять через GitHub Desktop

1. Установить GitHub Desktop.
2. Клонировать репозиторий.
3. Заменить изменённые файлы.
4. Проверить список изменений.
5. Написать сообщение коммита.
6. Нажать `Commit to main`.
7. Нажать `Push origin`.
8. Дождаться Deploy на Render.

## 8. Проверка после обновления

### Render

Проверить:

- статус `Live`;
- отсутствие ошибок сборки;
- отсутствие ошибок запуска Gunicorn;
- последние Application Logs.

### Сайт

Проверить:

- главную страницу;
- dashboard;
- форму поиска;
- каждый источник отдельно;
- мобильную версию;
- сообщения об ошибках.

## 9. Правила изменения кода

- Не создавать `app_fixed.py`.
- Не менять имя главного Flask-приложения.
- Не удалять рабочие маршруты без согласования.
- Не сохранять токены в репозитории.
- Не отключать проверку OAuth `state`.
- Не скрывать ошибки без логирования.
- Не утверждать, что API работает, пока не получен реальный успешный ответ.
- Не менять сразу весь проект, если достаточно двух файлов.

## 10. Работа с архивами

Имя исходного архива:

```text
ai-career-agent-source-YYYY-MM-DD.zip
```

Имя исправленного архива:

```text
ai-career-agent-v0.X.Y-YYYY-MM-DD.zip
```

Нельзя хранить внутри архива:

- `.env`;
- реальные API-ключи;
- access tokens;
- refresh tokens;
- временные файлы;
- `__pycache__`;
- виртуальное окружение.

## 11. Начало нового чата

В начале нового рабочего чата писать:

```text
Работаем над AI Career Agent.
Используй PROJECT_GUIDE.md и ARCHITECTURE.md.
Прикладываю актуальный архив GitHub.
Сначала проверь структуру и только потом меняй код.
```

## 12. Завершение задачи

После успешной проверки:

1. обновить `CHANGELOG.md`;
2. при необходимости изменить номер версии;
3. обновить `ROADMAP.md`;
4. сохранить новый стабильный ZIP;
5. сделать GitHub Release для значимой версии.
