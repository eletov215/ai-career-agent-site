# AI Career Agent — правила версий

Проект использует семантическое версионирование:

```text
MAJOR.MINOR.PATCH
```

Пример:

```text
0.3.2
```

## MAJOR

Меняется при крупной несовместимой переработке.

Пример:

```text
1.0.0 → 2.0.0
```

## MINOR

Меняется при добавлении новой функции без разрушения старых.

Пример:

```text
0.2.0 → 0.3.0
```

## PATCH

Меняется при исправлении ошибки.

Пример:

```text
0.3.0 → 0.3.1
```

## Этапы проекта

Пока продукт находится в разработке, используется версия `0.x.y`.

Предлагаемая схема:

```text
0.1.0 — базовый Flask-сайт
0.2.0 — поиск Trudvsem
0.3.0 — единый поиск по провайдерам
0.4.0 — подключение аккаунтов
0.5.0 — анализ вакансий
0.6.0 — профиль и резюме пользователя
0.7.0 — система сохранённых вакансий
0.8.0 — подготовка ответов на вопросы
0.9.0 — предрелизная версия
1.0.0 — стабильный MVP
```

## Теги Git

Для стабильных версий создавать тег:

```text
v0.3.0
```

## Имена архивов

```text
ai-career-agent-v0.3.0.zip
```

## Релиз

Перед релизом:

1. проверить Python-синтаксис;
2. проверить запуск;
3. проверить Render;
4. проверить основные маршруты;
5. обновить `CHANGELOG.md`;
6. обновить номер версии;
7. создать Git tag;
8. при необходимости создать GitHub Release.
