# История изменений AI Career Agent

Формат основан на Keep a Changelog.

## [Unreleased]

### Исправлено

- Кнопка ручного обновления «Работы России» теперь гарантирует запуск живого фонового worker.
- Страница отслеживает состояние синхронизации через `/trudvsem/status` и автоматически обновляет список после завершения.
- Ошибка фоновой синхронизации отображается пользователю вместо незаметного возврата к старому кэшу.

### Исправлено — устойчивость Trudvsem

- Увеличен размер одной пачки синхронизации до 100 вакансий, поэтому для лимита 300 требуется до 3 запросов вместо 30.
- Добавлены повторные попытки при тайм-аутах, временных сетевых ошибках и ответах 429/5xx.
- Тайм-аут чтения увеличен до 30 секунд.
- Синхронизация больше не закрывает HTTP-соединение после каждого запроса.
- Пользователь получает понятное сообщение о временной недоступности API без технической трассировки.
- Интерфейс дольше ожидает завершения медленного обновления и поясняет, что кэш после нового деплоя Render может быть пустым.

### Планируется

- Исправить или обойти причину `403` для открытого поиска HH только официальным способом.
- Убрать OAuth-коды и чувствительные параметры из логов.
- Не передавать пустой `text=` в запрос HH.
- Продолжить развитие Trudvsem.
- Добавить новые источники вакансий.
- Улучшить мобильный интерфейс.

## [0.3.0] — 2026-07-20

### Добавлено

- Единая страница поиска вакансий.
- Поддержка выбора источника.
- Провайдер HeadHunter.
- Повторный публичный запрос после `401` или `403`.
- Диагностическое логирование ответа HH.
- Отображение ошибок источника на странице.

### Изменено

- HeadHunterProvider заменён с заглушки на реальный HTTP-запрос.
- Провайдер HH подключён к рабочему `app.py`.
- Render продолжает запускать `gunicorn app:app`.

### Известные проблемы

- `GET https://api.hh.ru/vacancies` возвращает `403 Forbidden`.
- Запрос с OAuth и запрос без OAuth возвращают одинаковую ошибку.
- В ответе присутствует `Server: ddos-guard`.
- Причина блокировки не подтверждена поддержкой HH.

## 2026-07-20 — Trudvsem small-batch sync

- Limited Trudvsem API pages to 10 vacancies per request.
- Changed the default sync batch from 100 to 10.
- Increased the read timeout to 45 seconds.
- Kept incremental saving after every successful batch, so a later network error does not discard already downloaded vacancies.
- Fixed the README mismatch with the real application default.
